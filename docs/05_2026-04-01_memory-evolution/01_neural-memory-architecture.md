# 下一代记忆系统：神经记忆架构设计脑暴

> 2026-04-01 设计探索——分析当前记忆系统的本质局限，借鉴 LLM 前沿技术，构想真正对齐神经科学的记忆架构。

## 1. 问题定义：当前记忆系统的本质局限

现有四层记忆（Working / Episodic / Semantic / Procedural）在概念层面对齐了海马体-皮层记忆模型，但实现本质是**结构化文本 + 程式化检索**：

- 检索是符号匹配（关键词 / 标签 / 时间范围），不是 pattern completion
- 设计文档中规划的"Hopfield 网络式联想回忆"未落地
- "向量相似度 + 时间邻近 + 情绪权重混合检索"未落地
- 每个认知周期调用所有 memory provider，属于过度检索

核心矛盾：**设计对齐了神经科学的"是什么"，但实现用的是数据库的"怎么做"**。

## 2. LLM 内部已有的四种记忆形态

| 记忆形态 | 载体 | 特性 | 生物对应 |
|---------|------|------|---------|
| 参数记忆 | 模型权重 | 慢写入、泛化、不可寻址 | 皮层长期记忆 |
| 上下文记忆 | Context Window | 快速、精确、容量有限、会话结束即消失 | 工作记忆 |
| KV Cache | Attention K/V 矩阵 | 压缩隐式、影响后续生成 | 丘脑中继 / 短期激活痕迹 |
| 外部记忆 | RAG 数据库 | 持久精确、需显式检索 | 海马体→皮层回忆链路 |

现有框架只利用**上下文记忆 + 外部记忆**，完全没碰**参数记忆**和 **KV Cache**。这意味着我们只用了 LLM 四种天然记忆形态中的两种。

## 3. 两个候选方案

### 方案一：LoRA 微调索引 + SQLite 全量存储

核心思路：LoRA 不存记忆本身，而是学会**"怎么找记忆"**——对应海马体的索引功能。SQLite 承担皮层的长期存储角色。

**优势**：

- 可解释、可审计——记忆内容始终在 SQLite 中以明文存在
- 支持精确增删改——满足合规和隐私要求
- Adapter 体积小（几十 MB），可频繁训练
- 失败安全——LoRA 索引失败可 fallback 到传统检索

**劣势**：

- 需要足够数据量才能训练有效的索引模型
- 索引与存储的一致性维护
- 冷启动问题——新 Agent 没有足够 episode 来训练

### 方案二：运行时微调进端侧小模型（Llama/Qwen-8B）

核心思路：记忆 = 模型参数，最接近生物范式。记忆的存储、检索、泛化全部发生在参数空间。

**优势**：

- Pattern completion 天然存在——给部分线索即可激活完整记忆
- 语义泛化天然——相似情境自动触发相关经验
- 遗忘可作为特性——旧的不重要记忆自然被覆盖

**劣势**：

- 灾难性遗忘是双刃剑——重要记忆也可能被覆盖
- 微调成本——8B QLoRA 需 ~16GB VRAM
- 不可解释、不可编辑——无法精确删除特定记忆

## 4. 从 LLM 领域借鉴的六项关键技术

### 4.1 kNN-LM：参数预测与实例预测实时插值

参数化知识和显式记忆不是串行管线，而是**实时融合**——模型在每个 token 位置同时产生参数预测和 kNN 检索预测，通过学到的插值系数加权合并。

与 NeuroCore Global Workspace 的 competition 机制天然吻合：不同记忆源的 proposal 在工作空间中竞争广播权。

### 4.2 Self-RAG：学会"何时回忆"和"回忆什么"

不是每个 cycle 都检索，而是学会判断**是否需要调动记忆**。当前系统每 cycle 调用所有 provider 是过度检索——Self-RAG 的 retrieval token 机制可以让系统自主决定何时、从哪个源检索。

### 4.3 LoRA 组合与 Adapter Bank：模块化记忆

每个记忆领域 / 技能域是独立的小 LoRA adapter，运行时动态路由激活，类似 MoE gating 机制：

- **巩固** = 蒸馏进同一 adapter
- **遗忘** = 剪枝低效用 adapter
- **技能迁移** = 复用旧 adapter 部分权重

可与 NeuroCore 的 Skill 体系直接对接——每个 Skill 背后可以关联一组 adapter。

### 4.4 Soft Prompt / Prefix Tuning：记忆的最紧凑表征

巩固后的语义记忆表达为几个 soft token（几百字节），检索 = 选择激活哪些 soft prompt prefix。这是记忆从"具体事件"到"抽象知识"的最紧凑蒸馏形式。

### 4.5 KV Cache 压缩与持久化

H2O 等工作表明 KV Cache 大部分冗余，保留压缩后的 heavy hitter KV 对可实现真正的工作记忆持久化。比序列化文本再塞 prompt 高效得多——直接在注意力层面保持激活状态。

### 4.6 EWC（弹性权重巩固）：突触巩固的计算实现

保护"重要记忆"对应的参数，让不重要记忆自然被覆盖。重要性度量来自 episode 的 valence（情绪价值）和 outcome（结果反馈）——高情绪、高影响的经历获得更强的参数保护。

## 6. 三个核心设计原则

### 原则一：记忆形态随成熟度流动

```
原始 episode  →  soft prompt  →  LoRA adapter  →  模型权重
  (具体)        (半泛化)        (领域技能)       (深层直觉)
  (大)          (几百字节)      (几十 MB)        (不可分离)
  (可删除)      (可删除)        (可插拔)         (永久)
```

不是所有记忆都走到终点。大部分停在 episode 或 soft prompt 层——只有反复出现、高价值的经验才会逐步巩固为更深层的记忆形态。

### 原则二：检索不是查询，是激活

不是 `SELECT WHERE`，而是**神经激活扩散**——当前 context 自动激活相关 adapter、soft prompt 和 episode。不同记忆形态并行、竞争、互相增强，最终由 Global Workspace 的 competition 机制决定哪些记忆进入意识。

### 原则三：遗忘是特性，不是 bug

- EWC Fisher 信息量自然衰减
- Adapter bank 容量上限触发最低效用淘汰
- Episode store TTL 自动过期

遗忘自然发生。重要记忆因频繁激活而持续巩固，不重要的自然消退——这与生物记忆的遗忘曲线一致。

## 7. 待探讨问题

| 问题 | 选项 | 考量 |
|------|------|------|
| 端侧小模型定位 | 纯记忆体 vs 辅助推理（双 LLM 架构） | 纯记忆体更简单但浪费推理能力；辅助推理引入协调复杂度 |
| 微调时机 | 实时 vs 周期性批量（"睡眠巩固"） | 实时延迟高但记忆即时可用；批量效率高但有延迟窗口 |
| 硬件约束 | GPU（~16GB VRAM）vs 纯 CPU | 8B QLoRA 需 ~16GB VRAM，纯 CPU 场景可能需要更小模型或纯 soft prompt 方案 |
