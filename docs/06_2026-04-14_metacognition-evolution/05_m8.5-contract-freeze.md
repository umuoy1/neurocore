# M8.5 Contract Freeze

> 2026-04-15
> 作用：冻结元认知控制平面的关键对象语义，避免 `FastMonitor / DeepEvaluator / MetaController / future ControlAllocator` 再次出现语义分叉。

## 1. 当前冻结结论

M8.5 的第一阶段目标不是继续扩模块，而是先把“最终控制决策”收敛成单一路径。

本轮收口后，以下对象的语义固定如下：

### `FastMetaAssessment`

- 定位：快层监测输出
- 作用：表达 provisional confidence、trigger tags、是否触发 deep path、快层建议控制动作
- 不是最终控制决策

### `MetaAssessment`

- 定位：统一的深层评估结果
- 作用：表达 process / evidence / simulation / tool readiness / conflict / controllability 等综合评估结果
- 其中 `recommended_control_action` 是深评估建议，不是最终执行决策

### `VerificationTrace`

- 定位：deep path 的统一验证痕迹
- 当前唯一生产路径：`DeepEvaluator`
- 当前内容已切到 verifier / simulator 输出聚合，不再由 `DeepEvaluator` 手工拼整份 trace

### `CalibrationRecord`

- 定位：执行后校准样本
- 当前唯一写入路径：runtime 执行后经 `Calibrator.record(...)`

### `MetaDecisionV2`

- 定位：最终控制决策真源
- 当前唯一职责：表达 cycle 结束时真正采用的控制动作与其来源
- 这是当前唯一应该表达“最终控制决策”的对象

### 旧 `MetaDecision`

- 定位：legacy compatibility output
- 当前职责：给现有 `AgentRuntime` 和旧调用方提供兼容输出
- 不再允许重新计算一套最终控制结论

## 2. 唯一来源约束

以下字段从本轮开始约束唯一来源：

### `calibrated_confidence`

- 当前正式来源：`MetaAssessment.calibrated_confidence`
- `MetaDecisionV2.confidence` 应消费统一后的决策置信度
- 当前已完成 `fast/deep -> Calibrator.calibrate(...) -> MetaAssessment.calibrated_confidence -> MetaDecisionV2.confidence` 的单一路径收口

### `recommended_control_action`

- 快层来源：`FastMetaAssessment.recommended_control_actions`
- 深层来源：`MetaAssessment.recommended_control_action`
- 最终执行来源：`MetaDecisionV2.control_action`

### `selected_action_id`

- 最终来源：`MetaDecisionV2.selected_action_id`
- 旧 `MetaDecision.selected_action_id` 仅由 adapter 映射，不再自行计算

### `trigger_deep_eval`

- 唯一来源：`FastMetaAssessment.trigger_deep_eval`

### `verification_trace`

- 唯一来源：`MetaAssessment.verification_trace`
- 当前由 `DeepEvaluator` 聚合 `Verifier SPI + CounterfactualSimulator SPI` 输出

## 3. 当前控制平面结构

当前主链已调整为：

```text
MetaSignalBus
  -> FastMonitor
  -> DeepEvaluator | Fast fallback MetaAssessment
  -> DefaultControlAllocator
  -> MetaDecisionV2
  -> DefaultMetaController(adapter only)
  -> legacy MetaDecision
```

这意味着：

- `cycle-engine` 已经切到 allocator 主导
- `meta-controller.ts` 在正常路径下只做兼容映射
- trace 中可同时看到：
  - `fast_meta_assessment`
  - `meta_assessment`
  - `meta_decision_v2`

## 4. 当前仍未冻结完成的部分

以下事项还没有完全收口，留给 M8.5 后续阶段：

- `MetaSignalBus` 仍是单体聚合器，尚未 family-provider 化
- `ReflectionLearner` 仍未落库
- provider-level calibration profile 仍未实现
- verifier 级隔离、熔断与预算策略还是第一版

## 5. 本轮验收口径

本轮 M8.5 Phase 0 / 1 / 2 的验收口径为：

- 不再存在两个不同对象同时表达最终控制决策
- `MetaDecisionV2` 成为最终控制动作的唯一真源
- `DefaultMetaController` 不再在正常主链里私自重算一套最终控制决策
- `CycleTraceRecord` 能追踪到 `meta_decision_v2`
- `Calibrator` 成为正式校准入口，支持决策前查询与决策后写回
- `DeepEvaluator` 不再内置私有 heuristic calibration
- `DeepEvaluator` 已切到 `Verifier SPI` 编排层，并支持并发执行、部分失败降级和 budget-aware 选择
- 至少一个持久化 store 能保证 calibration 数据重启不丢
